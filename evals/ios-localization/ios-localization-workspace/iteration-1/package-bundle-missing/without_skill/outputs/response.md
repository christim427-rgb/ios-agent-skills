# Swift Package Localized Strings Not Appearing

## The Problem

```swift
// Inside MyPackage/Sources/FeatureAuth/LoginView.swift
Text("login_title")
String(localized: "login_subtitle")
```

These show the key ("login_title") instead of the translated text because the localization lookup is searching in the wrong bundle.

## Root Cause

By default, both `Text()` and `String(localized:)` look for translations in `Bundle.main` -- the main application bundle. However, your localization files are inside the Swift Package's own bundle, not the main app bundle.

## The Fix

You need to specify `bundle: .module` so the lookup targets the package's resource bundle:

```swift
// SwiftUI Text
Text("login_title", bundle: .module)

// String(localized:)
String(localized: "login_subtitle", bundle: .module)
```

`.module` is a static property automatically generated by SPM when the package has resources (including localization files).

## Additional Requirement: Package.swift Configuration

Your `Package.swift` must declare `defaultLocalization`:

```swift
let package = Package(
    name: "MyPackage",
    defaultLocalization: "en",  // Required for localized resources
    // ...
    targets: [
        .target(
            name: "FeatureAuth",
            resources: [
                .process("Resources")  // Where your .lproj folders live
            ]
        )
    ]
)
```

Without `defaultLocalization: "en"` in Package.swift, SPM will not process localization resources.

## Directory Structure

Make sure your `.lproj` directories are inside the package's resources folder:

```
Sources/
  FeatureAuth/
    LoginView.swift
    Resources/
      en.lproj/
        Localizable.strings
      es.lproj/
        Localizable.strings
```

## Checklist

1. Add `defaultLocalization: "en"` to `Package.swift`
2. Place `.lproj` directories in the target's resources folder
3. Add `bundle: .module` to all `Text()`, `String(localized:)`, and `NSLocalizedString()` calls within the package
